# ADR 008: Stateless JWT Authentication

## Status
Accepted

## Context
The platform requires a secure, scalable way to authenticate API requests without hitting the database for every request to validate a session.

## Decision
We use JSON Web Tokens (JWT) for authentication. Tokens are signed with a secure secret and contain essential identity information (userId, tenantId).

## Consequences
- **Positive:** Stateless, easy to scale horizontally. Fast verification.
- **Negative:** Harder to invalidate tokens immediately (requires token blacklisting or short expiration times).
